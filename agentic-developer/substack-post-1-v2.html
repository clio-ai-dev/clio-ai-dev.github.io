People think AI coding means autocomplete.

Tab, tab, tab.

That was 2024.

Here's what AI agents are actually doing for me. Today. Every day. At work and at home.

And no, this isn't a demo. This is my real workflow.

Understanding a massive codebase in minutes

I was handed an ambiguous requirement at work. The kind where you read it three times and still don't know what to build.

Our codebase has 100+ projects.

Instead of spending half a day reading through code, I asked my AI agent to explain the requirement in the context of our entire codebase.

It understood exactly what the requirement was about. It found the relevant components. It told me precisely what to implement and where.

Time spent: about 5 minutes.

Debugging production errors I'd never seen before

My team noticed a bunch of errors in our ASP.NET Core service in production. Kusto logs showed something was off.

Problem: I was totally new to this service. I didn't know how we log things. I didn't know the patterns.

I asked my agent to connect to Kusto via MCP, find the errors, find the patterns, and figure out what's going on.

It found the exact pattern. Correlated it with the codebase. Told me exactly what was broken and what to fix.

I would have needed a full day to ramp up. The agent did it in one conversation.

Never failing a CI pipeline again

I used to create my pull request, push it, and then wait. Sometimes an hour.

Only to find out the ADO pipeline failed because:

  • It builds the code for release (different flags than my local box)
  • It checks code styling differently
  • It runs unit tests I forgot to run

Then I'd fix it, push again, wait again. Easily 2 hours lost.

I added custom instructions to the repo. Now my agent knows exactly what process to run on my machine before creating the PR.

Result: PRs pass on the first try. Every time.

PRs that write themselves

I used to forget to add a good description to my pull requests. Then I'd have to manually reply to review comments one by one.

Now I ask the agent to create the PR with a detailed description based on the work we did together.

When reviewers leave comments, the agent reads them, addresses the feedback, pushes the fix, replies to the comments, and resolves them.

I just approve the agent's work.

3 implementations, 3 PRs, 0 meetings

I needed to propose 3 different approaches for a small feature to my team.

The old way: write a design doc, explain each approach, schedule a meeting, talk through slides.

The new way: I spun up 3 sub-agents. Each one implemented a different approach as a separate pull request.

The team looked at real code. They voted. No slides. No meetings. No "let me explain what I mean."

3 AI models, 1 consensus

I wanted to upgrade the Python toolchain for my repo. My agent modified a bunch of files for the upgrade.

But I wasn't convinced it did the best job with just one model.

So I spun up 3 sub-agents: Claude, Codex, and Gemini. Asked all three to independently figure out the actual upgrade requirements.

They reached consensus on what was needed.

When the first build failed with the new toolchain, I looped the agents again. They kept finding remaining issues until all three agreed nothing else was needed.

Multi-model consensus. Not something I could have done manually.

Provisioning my own AI assistant without touching a terminal

I wanted a personal AI assistant running 24/7. Not ChatGPT. An actual agent with access to my files, calendar, and tools.

Instead of figuring out how to set up a VM, I asked an AI agent to connect to my Azure subscription, provision a virtual machine, install the software, and tell me how to connect at the end.

I never ran a single command. Never opened the Azure portal.

From there, that assistant (her name is Clio) has:

  • Drafted 6 LinkedIn posts per week for my side business
  • Built an entire 32-concept visual guide with custom illustrations
  • Managed my calendar and created reminders for my family
  • Set up this newsletter you're reading right now
  • Built a lunar eclipse animation for my kids

All I do is review.
