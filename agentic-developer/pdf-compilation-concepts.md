# The Agentic Developer: Visual Archive — Concept List

**Format:** Each concept = 1 full-page visual/infographic + 1 page short description (~150-200 words). Think LinkedIn-post-style: punchy insight, visual proof, actionable takeaway.

**Target:** ~35 concepts = ~70 pages (attractive lead magnet, similar to ByteByteGo's "Big Archive")

---

## Part 1: The Shift (Concepts 1-5)

1. **The Three Eras of AI-Assisted Development**
   Autocomplete (2021) → Chat (2023) → Agents (2025). An ascending staircase showing what changed at each stage.

2. **The METR Paradox: The 39-Point Perception Gap**
   Developers think they're 20% faster but are actually 19% slower. Two bars showing perceived vs actual speed.

3. **Your New Job Description**
   Old job: 70% writing code. New job: 40% reviewing + 40% specifying + 20% coding. Side-by-side pie charts.

4. **The Discipline Gap**
   Same AI tool, opposite outcomes. Fork in the road: naive use (downhill) vs disciplined use (uphill).

5. **The 80/20 of Agentic Development**
   Three practices that drive 80% of productivity: context files, tests first, spec your tasks. Pyramid diagram.

## Part 2: Mindset (Concepts 6-10)

6. **The Junior Dev with Perfect Memory and Zero Judgment**
   AI knows everything on the internet but can't judge what's right for YOUR project. Robot with huge brain, tiny lightbulb.

7. **The Director Mindset**
   You're not a coder anymore, you're a director. Vision, constraints, quality bar. Film director metaphor.

8. **The Five Mental Traps**
   Over-trusting, under-specifying, anthropomorphizing, all-or-nothing, static calibration. Five traps on a path.

9. **From Vibe Coding to Vibe Engineering**
   The spectrum: Vibe Coding (10%) → Vibe Engineering (50%) → Agentic Engineering (30%) → Traditional (10%).

10. **When to Use Which Mode**
    Decision flowchart: blast radius, lifespan, team touch. Quick visual guide for choosing your approach.

## Part 3: Context Engineering (Concepts 11-18)

11. **The Context Stack**
    Layered system: codebase → tests → AGENTS.md → path rules → skills → MCP. Where to invest effort.

12. **Without Context vs With Context**
    Same prompt, wildly different output. Split panel showing generic code vs architecture-matching code.

13. **Context File Anti-Patterns**
    Four failure modes: The Novel, The Museum, The Wishlist, The Secret Keeper. 2x2 grid.

14. **How Path-Scoped Rules Work**
    Project tree showing rules that activate only when specific files are edited. Surgical context.

15. **The Reactive Rule Workflow**
    Agent makes mistake → you correct → you add a rule → agent never repeats it. Feedback loop.

16. **Skills: From Commands to Intent**
    Old: memorize slash commands. New: describe intent, agent finds the right skill. The discovery upgrade.

17. **What MCP Is (and Isn't)**
    Hub-and-spoke: MCP connects to live data sources. Static context = what you knew. MCP = what's true now.

18. **Designing an AI-Friendly Codebase**
    Small files, clear boundaries, consistent patterns. Checklist of what makes code easy for agents to work with.

## Part 4: The TDD Agent Loop (Concepts 19-23)

19. **The TDD Agent Loop**
    Circular flow: Write failing test → Agent implements → Tests pass → Refactor → Repeat. The signature workflow.

20. **The Confidence Equation**
    Three dials: specification clarity + test coverage + context quality = confidence in agent output.

21. **Why Tests Come First (Not After)**
    Tests as specification, not verification. Tests tell the agent WHAT to build, then verify it did.

22. **The Review Checklist**
    What to check in agent output: correctness, security, performance, style, architecture fit. Annotated code review.

23. **When to Reject and Re-prompt**
    Decision tree: small fix → edit yourself. Architectural miss → reject, improve context, re-run.

## Part 5: Advanced Patterns (Concepts 24-30)

24. **Multi-Agent Orchestration**
    Multiple specialized agents working together: one for code, one for tests, one for docs. Assembly line.

25. **The Worktree Pattern**
    Parallel git worktrees: multiple agents working on different features simultaneously without conflicts.

26. **Agent-Driven CI/CD**
    Pipeline where agents run tests, fix failures, update docs, and deploy. The fully automated loop.

27. **Always-On Agents**
    Agents that watch your repo 24/7: auto-triage issues, suggest fixes, maintain docs. The night shift.

28. **Security in an Agentic World**
    Principle of least privilege for AI agents. Sandboxing, token scoping, audit trails. Lock diagram.

29. **Cost Management**
    Token usage patterns, caching strategies, knowing when NOT to use an agent. Dollar signs and dials.

30. **The Human in the Loop**
    Where human judgment is irreplaceable: architecture decisions, security review, UX, ethics. Guard rail diagram.

## Part 6: The Future (Concepts 31-35)

31. **The Anthropic Forecast**
    3-6 months: most code AI-written. 12 months: AI handles entire features. What this means for you.

32. **The Skills That Will Matter**
    Communication, system design, context engineering, review discipline. The new developer skill stack.

33. **Building Your Agentic Toolkit**
    Essential tools: IDE + agent, context files, MCP servers, CI/CD integration. The modern developer's workbench.

34. **The Compound Effect**
    Each context file, each rule, each test makes every future agent session better. Exponential curve.

35. **From Individual to Team**
    Scaling agentic practices across a team: shared AGENTS.md, team rules, collective context. Network effect.
