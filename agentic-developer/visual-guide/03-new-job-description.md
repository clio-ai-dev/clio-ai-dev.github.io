# 03 — Your Job Description Just Changed

![Old Job vs New Job](../images/img-03-01.jpg)

Two years ago, a typical senior developer spent roughly 70% of their time writing code. Reading docs, debugging, meetings, and code review split the rest.

That ratio has flipped.

In the agentic era, your time breaks down closer to:

- **40% Reviewing agent output.** Reading, verifying, testing what the agent wrote. This is now your primary skill.
- **40% Specifying and providing context.** Writing clear specs, maintaining context files, crafting precise task descriptions. The better your input, the less time you spend reviewing.
- **20% Writing code by hand.** The parts that are too nuanced, too security-critical, or too architecturally important to delegate.

This isn't a demotion. It's a promotion.

You went from individual contributor to engineering manager, except your direct reports are AI agents that work at machine speed, never take PTO, and have read every Stack Overflow answer ever written.

The catch? They have zero judgment. They'll confidently implement the wrong architecture, use deprecated libraries, and introduce security vulnerabilities if you don't specify otherwise.

Your new job is making sure they don't.

**Writing code was the old job. Directing code is the new one.**
