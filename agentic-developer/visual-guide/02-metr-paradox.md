# 02 — The METR Paradox: Developers Think They're 20% Faster But Are 19% Slower

![The 39-Point Perception Gap](../images/img-02-01.jpg)

Here's the most uncomfortable finding in AI-assisted development.

METR, an independent research lab, ran a controlled study with experienced open-source developers working on their own repositories. The result? Developers using AI agents were **19% slower** than without them.

But here's the real kicker: those same developers *estimated* they were **20% faster**.

That's a 39-point perception gap. Nearly four in ten percentage points of pure self-deception.

Why does this happen? Three reasons:

1. **The speed illusion.** Watching an agent generate 200 lines in seconds *feels* fast. But you still have to read, understand, verify, debug, and often rewrite those 200 lines.

2. **Hidden costs.** Time spent crafting prompts, waiting for responses, context-switching between "writing mode" and "reviewing mode," and debugging AI-generated bugs that you wouldn't have written yourself.

3. **The feedback loop is broken.** When you write code yourself, you know immediately if you're stuck. When an agent writes confidently wrong code, you might not realize for 30 minutes.

The fix isn't to stop using AI. It's to stop using it naively. The variable isn't the tool. It's the methodology.
