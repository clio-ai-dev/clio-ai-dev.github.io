## Part I: The Shift (Chapters 4-5)

**Chapter 4: The Agentic Engineering Mindset** — The mental models that separate productive agentic developers from frustrated ones. How to think about agents as junior developers with perfect memory and zero judgment.

**Chapter 5: From Vibe Coding to Vibe Engineering** — Karpathy's evolution from casual prompting to serious discipline, and what that trajectory teaches us about where the field is headed.

## Part II: Context Engineering (Chapters 7-10)

**Chapter 7: Path-Scoped Rules and When to Use Them** — Beyond the main context file. How to create modular, targeted rules for different parts of your codebase.

**Chapter 8: Skills: Teaching Your Agent New Tricks** — The skills system in Claude Code and how to create reusable, LLM-invokable instruction sets for complex workflows.

**Chapter 9: MCP Servers and External Tool Integration** — Connecting agents to databases, APIs, and custom tooling through the Model Context Protocol. Includes building an MCP server in C#.

**Chapter 10: Designing an AI-Friendly Codebase** — Architecture and code organization patterns that make agents dramatically more effective. The practices that were always good engineering but now have 10x payoff.

## Part III: The Tools (Chapters 11-12)

**Chapter 11: IDE Agents: Copilot, Cursor, and Windsurf** — When to use each, how to configure them, and the specific workflows where each excels.

**Chapter 12: CLI Agents: Claude Code and Codex CLI** — Terminal-native agent workflows for power users. Sub-agents, parallel execution, and scripting agent tasks.

## Part IV: The Workflows (Chapters 14-17)

**Chapter 14: Spec-First Development** — Write the design doc, break it into tasks, let agents implement. The workflow Simon Willison uses daily.

**Chapter 15: Error-Message-Driven Debugging** — Paste the error, get the fix. How to structure error context so agents diagnose accurately.

**Chapter 16: Codebase Archaeology and Documentation** — Using agents to understand and document existing code. Generating internal docs, architecture diagrams, and onboarding guides.

**Chapter 17: Migration Generation** — Framework upgrades, database migrations, API versioning. The tasks agents handle exceptionally well with the right context.

## Part V: Scaling Up (Chapters 18-22)

**Chapter 18: The Parallel Agent Lifestyle** — Running 3-5 agents simultaneously on different tasks. Git worktrees, task decomposition, and review orchestration.

**Chapter 19: Background Agents and Async Workflows** — Fire-and-forget from your phone. GitHub Copilot Coding Agent, Codex Cloud, and overnight automation.

**Chapter 20: Multi-Agent Coordination** — Agent teams working the same codebase. File-based locking, orchestration patterns, and conflict resolution.

**Chapter 21: Agent + CI/CD Integration** — Agents that react to failing builds, fix issues automatically, and manage deployment pipelines.

**Chapter 22: Cost Management and Token Economics** — Nobody talks about this. The real cost of running multiple agents, and how to optimize spend without sacrificing productivity.

## Part VI: .NET-Specific Patterns (Chapters 23-26)

**Chapter 23: Context Files for .NET Projects** — Templates for ASP.NET Core, Blazor, MAUI, and class library projects. The .NET-specific conventions agents need to know.

**Chapter 24: Agents and .NET Aspire** — Using the Aspire dashboard for agent telemetry, building cloud-native AI services, and the Aspire + agent development workflow.

**Chapter 25: EF Core Migrations with Agents** — Database schema evolution through agents. Migration generation, seed data, and schema review workflows.

**Chapter 26: Building Multi-Agent Apps with Microsoft Agent Framework** — Building your own multi-agent systems in .NET. The Agent Framework, Semantic Kernel integration, and production deployment.

## Part VII: The Discipline (Chapters 27-29)

**Chapter 27: Security Review with Agents** — OWASP Top 10 scanning, dependency auditing, secrets detection, and systematic security review workflows.

**Chapter 28: When NOT to Use Agents** — The tasks where agents hurt. How to recognize them, and the discipline to do the work by hand when that's the right call.

**Chapter 29: The 30-Day Transformation Plan** — Day-by-day plan to go from wherever you are today to a fully operational agentic workflow. Concrete actions, milestones, and checkpoints.
